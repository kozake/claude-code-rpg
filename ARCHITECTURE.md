## プロジェクト概要

PixiJS と Web Audio API のみで構築したブラウザ動作のターン制 2D ダンジョン RPG。

- 10 フロアの手続き生成ダンジョン
- ターン制戦闘（プレイヤー → 全敵の順番）
- レベルアップ・装備成長システム
- キーボード・タッチ・仮想ゲームパッド対応

## 技術スタック

| ライブラリ | バージョン | 用途 |
|---|---|---|
| [PixiJS](https://pixijs.com/) | 7.4.2 | WebGL/Canvas レンダリング |

## モジュール一覧と責務

### `constants.js` — 定数定義

ゲーム全体で参照する定数をまとめたファイル。変更時の影響範囲が広いため、バランス調整はここで行う。

```
TILE_SIZE = 40         // 1マスのピクセルサイズ
MAP_COLS = 20          // マップ横マス数
MAP_ROWS = 16          // マップ縦マス数
SCREEN_WIDTH = 800     // 画面幅
UI_HEIGHT = 200        // UI エリア高さ
SCREEN_HEIGHT = 800    // 合計画面高さ (640 + 160)
MAX_FLOORS = 10        // 総フロア数

TILE.FLOOR = 0
TILE.WALL  = 1
TILE.STAIRS = 2

ENEMY_DEFS  — 各敵の基本ステータス表（name, hp, attack, defense, xp, minFloor）
BOSS_DEF    — Dark Lord 専用定義
ITEM_DEFS   — 消費・武器・防具のアイテム定義
```

---

### `Game.js` — ゲームループ・ステートマシン

ゲームの中心クラス。すべてのシステムはここから呼ばれる。

**ゲームステート**

```
'title'    → タイトル画面
'opening'  → オープニングシーン
'playing'  → ゲームプレイ中
'gameover' → ゲームオーバー
'ending'   → ボス撃破エンディング
'win'      → 勝利画面
```

**主要メソッド**

| メソッド | 説明 |
|---|---|
| `_startGame()` | タイトル→プレイ移行の初期化 |
| `_initLevel()` | フロア生成・敵/アイテム配置・階段設置 |
| `move(dx, dy)` | プレイヤー移動・当たり判定・戦闘判定 |
| `_playerAttack(enemy)` | ダメージ計算・エフェクト呼び出し |
| `_enemyTurns()` | 全敵の AI 行動 |
| `_trySpecialAttack()` | 魂の一撃（ソウル 100 消費、3 倍ダメージ） |
| `_placeItems(rooms)` | 部屋にアイテムをランダム配置 |
| `_scaleEnemy(def)` | フロアに応じた敵ステータス倍率適用 |

**スケーリング式**

```
倍率 = 1.0 + (floor - 1) × 0.12
```

### `Player.js` — プレイヤーエンティティ

**基本ステータス**

| 項目 | 初期値 | 成長 |
|---|---|---|
| HP | 30 | レベルアップ +5 |
| ATK | 6 | レベルアップ +2 |
| DEF | 2 | レベルアップ +1 |
| ソウルゲージ | 0/100 | 被ダメージで増加 |

**XP テーブル**（倍率 1.5×）: 20 → 30 → 45 → 68 → ...

**主な機能**
- グリッド間の滑らかな移動トゥイーン
- 被ダメージ時のフラッシュエフェクト
- 浮遊アニメーション（bob）
- インベントリ（消費アイテムのみ、装備は自動装着）

---

### `Enemy.js` — 敵エンティティ

**敵種別**

| 名前 | 登場フロア | HP | ATK | DEF | XP |
|---|---|---|---|---|---|
| Slime | 1+ | 8 | 3 | 0 | 5 |
| Goblin | 1+ | 14 | 5 | 1 | 10 |
| Skeleton | 3+ | 20 | 7 | 2 | 15 |
| Orc | 3+ | 30 | 9 | 3 | 22 |
| Dark Knight | 5+ | 45 | 14 | 5 | 38 |
| Dark Lord (ボス) | 10 | 350 | 32 | 12 | 500 |

**ボス固有能力**
- HP が 50% 以下でフェーズ 2 移行（ATK +10）
- フェーズ 2 では 2〜3 ターンに 1 回「呪いの一撃」を使用
  - 式: `max(2, floor((boss.ATK - floor(player.DEF / 2)) × 1.5))`

**AI 行動**
- プレイヤーとのマンハッタン距離 ≤ 8 → 追跡
- 距離 > 8 → ランダム移動
- 隣接時 → 攻撃

### `Dungeon.js` — ダンジョン生成

**マップ仕様**: 20 列 × 16 行、タイルサイズ 40px

**生成アルゴリズム（BSP 簡易版）**

```
1. 4〜7 マス幅・高さの部屋をランダム配置（最大 10 部屋）
2. 部屋同士を L 字廊下で接続（横→縦）
3. 最後の部屋に階段を配置
4. 松明・スカルデコレーションを各部屋に追加
```

生成失敗時のフォールバックとして、2 部屋構成のレイアウトを保証する。

**アイテム種別**

| 消費 | Elixir | HP 全回復 | 1 |

### `Audio.js` — サウンドシステム

BGM と SE をすべて Web Audio API で合成。外部ファイル不要。

**BGM 仕様**

- キー: D ナチュラルマイナー
- BPM: 76（ダークでゆっくりめ）

**声部構成**: メロディ・ハーモニー・ウォーキングベース・パッド

- 右下: ✨ ソウルボタン、アイテムボタン 1/2

## ゲームループとステート管理

```
[起動]
  ↓
[title] → クリック/タップ
  ↓
[opening] → テキスト表示完了 or スキップ
  ↓
[playing] ─────────────────────────────────────────────┐
  │                                                     │
  ├─ プレイヤー入力                                      │
  │    移動 → 衝突判定 → 敵接触 → _playerAttack()        │
  │    ソウル攻撃 → _trySpecialAttack()                  │
  │    アイテム使用 → インベントリから消費                │
  │                                                     │
  ├─ 敵ターン (_enemyTurns)                              │
  │    各敵: AI 判断 → 移動 or 攻撃                      │
  │                                                     │
  ├─ ダメージ解決                                        │
  │    ダメージ = max(1, ATK - DEF)                      │
  │    死亡 → XP 付与 → レベルアップ判定                 │
  │                                                     │
  ├─ アニメーション更新（60fps ticker）                  │
  │    トゥイーン, ボブ, カメラ, パーティクル             │
  │                                                     │
  └─ UI 更新                                            │
       バー・テキスト・メッセージキュー                   │
                                                        │
  HP = 0 → [gameover]                                   │
  階段到達 → floor++ → _initLevel() ──────────────────── ┘
  ボス撃破 → [ending] → [win]
```

## 戦闘システム

### ダメージ計算

```
通常攻撃ダメージ = max(1, 攻撃者.ATK - 防御者.DEF)
ソウル攻撃ダメージ = max(1, 攻撃者.ATK × 3)      // ソウル 100 消費
呪いの一撃ダメージ = max(2, floor((boss.ATK - floor(player.DEF / 2)) × 1.5))
```

### ソウルゲージ

- 被ダメージした分だけ増加（上限 100）
- 100 になると「魂の一撃」が使用可能
- 使用後はゼロリセット

### レベルアップ

- XP テーブル: `xpToNext *= 1.5`（20 → 30 → 45 → 68 → ...）
- 効果: HP +5（最大値も増加）、ATK +2、DEF +1

## ダンジョン生成

```
_generateRooms():
  for 試行回数 in range(20):
    w = rand(4, 7), h = rand(4, 7)
    x = rand(1, MAP_COLS - w - 1)
    y = rand(1, MAP_ROWS - h - 1)
    重複チェック（1マスバッファ）
    OK → rooms に追加（最大 10 部屋）

_connectRooms(rooms):
  隣接部屋のセンター同士を L 字で接続
  横方向→縦方向の順でフロアタイルを描画

階段: 最後の部屋のランダム位置に設置
```

## データ構造

### マップ配列

```javascript
// map[row][col] → TILE 定数 (0=FLOOR, 1=WALL, 2=STAIRS)
map: number[][]  // 16行 × 20列
```

### プレイヤー状態

```javascript
{
  gridX, gridY,           // グリッド座標
  hp, maxHp,
  attack, defense,
  level, xp, xpToNext,
  soul, maxSoul,          // ソウルゲージ
  inventory: string[],    // 消費アイテムキー（最大 2）
  weapon: string | null,  // 装備中武器
  armor: string | null,   // 装備中防具
}
```

### 敵状態

```javascript
{
  gridX, gridY,
  def: EnemyDef,   // constants.js の定義参照
  hp: number,
  alive: boolean,
  attackCount: number,  // ボスのフェーズ管理用カウンタ
}
```

## レンダリングパイプライン

```
PIXI.Application
  └─ stage (Container)
       ├─ dungeonContainer    ← タイル・松明・デコレーション
       ├─ itemContainer       ← アイテムスプライト
       ├─ enemyContainer      ← 敵スプライト・HP バー
       ├─ playerContainer     ← プレイヤースプライト
       ├─ effectsContainer    ← パーティクル・ダメージ数字
       └─ uiContainer         ← HUD・ステータス（固定位置）
```

カメラはプレイヤー座標を中心に `stage` のオフセットを計算して追従する（マイクロオフセットで滑らかに）。UI コンテナはカメラに追従しない固定レイヤー。
